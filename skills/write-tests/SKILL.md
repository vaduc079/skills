# Instructions

Analyze the code and provide a single unit test with the necessary imports, without any additional explanations or comments, unless absolutely necessary. Avoid repeating imports and mocks that exist in the codebase already.

If provided with specific instructions or asked to test a particular part or scenario, please follow those instructions and generate the test accordingly.

# Test Guidelines

- **IMPORTANT**: Test behavior, not implementation
- One assertion per test when possible
- Clear test names describing scenario
- Use existing test utilities/helpers
- Tests should be deterministic
